# Sistema de processamento de pagamento
## Foco: Interfaces e contratos
### Aprendizados
- Interface é quem define o "contrato" que diz o que deve ser feito e as classes é quem assina o contrato, é obrigada a implementar tudo que está no contrato mas decide como fazer.
- Uma classe que implementa uma interface deve ter todos os métodos ali escritos.
- Isso tudo é importante para garantir baixo acoplamento, ou seja, módulos interagirem pouco entre si, uma mudança não deve alterar o funcionamento do sistema.
